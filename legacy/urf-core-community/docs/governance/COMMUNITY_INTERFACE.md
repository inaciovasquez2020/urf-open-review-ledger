# Community Interface

## Canonical source
The canonical theorem spine, dependency ledger, closure claims, and axioms live only in `urf-core`.

## Admissible contribution classes
1. Examples
2. Expository maps
3. Formalization extensions that do not alter canonical claims
4. Tests
5. Implementations
6. Visualization and educational tooling

## Inadmissible contribution classes
1. Edits to canonical theorem statements
2. Edits to dependency ledger
3. Edits to closure-status claims
4. Edits to canonical axioms
5. Reframing the authoritative line

## Transfer rule
A contribution may be promoted into the canonical repository only by maintainer-controlled review and explicit re-entry commit.
